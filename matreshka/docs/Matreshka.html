Adds dependencies between properties

Arguments

keys2 string Array.<string> From what properties are dependent
getter <optional>

setOnInit <optional>
 		boolean true Pass false if you don't want to set dependency immediately
Since:

Source:
matreshka-core.js, line 1178
Example

Basic usage

this.a = 3;
this.b = 4;
this.addDependency( 'perimeter', 'a b', function() { return ( this.a + this.b ) * 2} );

this.on( 'change:perimeter', function() {
	alert( 'perimeter is changed to ' + this.perimeter );

bindElement(key, el, binder, evtOpts) → {mk}

Creates event bridge between DOM node and Matreshka instance property

This powerful function binds to each other element and instance property with given options that say "when and how to extract element's value", "how to set element's value when property is changed".
Returns: mk (self)
Fires: bind
Arguments
key string mk A key (or space-delimited list keys) that has to be bound to given element(s)
el Array.<Node> NodeList Node jQuery balalaika string An element (DOM Node or DOM NodeList or array of nodes or balalaika array or jQuery instance or css selector...) that has to be bound to given key(s). Pay attention that element is required, otherwise the error will be thrown.
binder <optional>
 		binder A binder object which contains following properties: setValue (how to set value for an element), getValue (how to extract value from an element), on (when we have to extract a value from an element and assign it to given property) and initialize (what to do before binder initialized). Look at the binder documentation for more info.
evtOpts <optional>
 		eventOptions If you want to set "silent" flag or pass some options to a "bind" event handler
Source:
matreshka-core.js, line 463
Examples

Basic usage 1

this.bindElement( 'myKey', 'input[type="checkbox"]', {
	on: 'click',
	getValue: function() {
		return this.checked; // "this" is checkbox element  
	},
	setValue: function( v ) {
		this.checked = !!v;
	}
});
// now when element is bound you can change myKey and look at changes in UI
this.myKey = true; // makes checkbox checked
this.myKey = false; // makes checkbox unchecked

Basic usage 2. Binding using Matreshka.defaultBinders array that contains binder for few types of DOM elements (input[type="text"], input[type="radio"], input[type="checkbox"], select, textarea). So you don't need to pass binder for such elements

this.bindElement( 'myKey', '.checkbox' );

Custom checkbox 1. This example shows how to create your own custom checkbox that has "checked" class if it's state is checked.

this.bindElement( 'myKey', '.custom-checkbox', {
	on: 'click',
	getValue: function() {
		return $( this ).hasClass( 'checked' );
	},
	setValue: function( v ) {
		$( this ).toggleClass( 'checked', !!v );
	}
});

Custom checkbox 2. In this example we'll do the same as in previous example but using predefined binder via Matreshka.defaultBinders.

//shift means that we're adding new default binder to the beginning of MK.defaultBinders list
MK.defaultBinders.shift( function( element ) {
	if( $( element ).hasClass( 'custom-checkbox' ) ) return {
		on: 'click',
		getValue: function() {
			return $( this ).hasClass( 'checked' );
		},
		setValue: function( v ) {
			$( this ).toggleClass( 'checked', !!v );
		}
	};
});

this.bindElement( 'myKey', '.custom-checkbox' );

"bind" event

this.on( 'bind:myKey', function() { alert( 'ok!' ); });
this.bindElement( 'myKey', '.custom-checkbox' ); // alerts "ok!"

"bind" event options

this.on( 'bind:myKey', function() { alert( 'ok!' ); });
this.bindElement( 'myKey', '.custom-checkbox', {}, { silent: true } ); // no alert

Extending default binders. For example we're working with input[type="text"]. By default "on" property for this element contains "keydown" string. But we want to use "blur" event for the element that has been bound to myKey property

this.bindElement( 'myKey', '.custom-checkbox', { on: "blur" });

Binding this to the element. If you want to use context (sandbox) for bindings of elements contained in single element, you can pass this special argument value to the method

// you can use this.bindElement( '__this__', '.app' ); instead
this.bindElement( this, '.app' );
// this.$( '.my-element' ) takes element(s) from .app
this.bindElement( 'myKey', this.$( '.my-element' ) );

"Bound element is mising" error

vae $el = $();
this.bindElement( 'x', $el ); // ERROR because $el is empty
// always check is element existing
if( $el.length ) {
	this.bindElement( 'x', $el );
}

<static> defaultBinders :function function

Matreshka.defaultBinders is the array of functions that examine given element by given rules and returns binder if examination gone successfully. It's using when Matreshka#bindElement(1) method did not get third argument.

Source:
matreshka-core.js, line 1689
Examples

HTML5 input type=number

//shift means that we're adding new default binder to the beginning of MK.defaultBinders list
MK.defaultBinders.shift( function( element ) {
	if( element.tagName === 'input' && element.type === 'number' ) return {
		on: 'mouseup',
		getValue: function() {
			return this.value;
		},
		setValue: function( v ) {
			this.value = v;
		}
	};
});
this.bindElement( 'myKey', '.my-input-type-number' );

Custom checkbox

MK.defaultBinders.shift( function( element ) {
	if( $( element ).hasClass( 'custom-checkbox' ) ) return {
		on: 'click',
		getValue: function() {
			return $( this ).hasClass( 'checked' );
		},
		setValue: function( v ) {
			$( this ).toggleClass( 'checked', !!v );
		}
	};
});
this.bindElement( 'myKey', '.custom-checkbox' );
