Class: Array

Matreshka. Array

Matreshka Array class (Matreshka "collection"). Extends Matreshka.

new Array()

Andrey Gubanov

Examples

Basic usage

new MK.Array;

Passing length

// creates Matreshka.Array instance with length = 42
new MK.Array( 42 );

Passing elements

// creates Matreshka.Array instance with length = 2
new MK.Array( 'Hi', { a: 'b' } );

Inheritance

var MyClass = Class({
	'extends': MK.Array,
	constructor: function() {
		// calls MK.Array constructor with this context and given arguments
		MyClass.parent.constructor( this, arguments );
	},
	method: function() {}
});

initializeSmartArray() → {boolean}

Initializes "smart array"

This method is only needed when you're setting {@Matreshka.Array#itemRenderer} property after some items are added.
Returns: boolean
Since:
0.1
Source:
matreshka-array.js, line 502
Examples

Basic usage

var mkArray = new MK.Array;
// DOM is not changing because itemRenderer is not assigned yet
mkArray.push( ... );
mkArray.itemRenderer = function() { '
MyDiv
' };
// DOM is changing after initializeSmartArray execution
mkArray.initializeSmartArray();

When initializeSmartArray is not needed

var mkArray = new MK.Array;
// setting itemRenderer before adding any item to array
mkArray.itemRenderer = function() { '
MyDiv
' };
// DOM is changing after push, no need to use initializeSmartArray
mkArray.push( ... );

setItemMediator()

Sets function that transforms items

This method is using when you want to keep your items to be a certain type (string, number, object...). Pay attention that new mediator overrides Model property
Since:
0.1
Source:
matreshka-array.js, line 185
Example
 
var mkArray = new MK.Array( 1, 2, 3, 4, 5 );
mkArray.setItemMediator( function( value ) {
	return String( value );
});
mkArray.push( 6, 7 );
mkArray.unshift( true, {} );

console.log( mkArray.toJSON() ); // [ "true", "[object Object]", "1", "2", "3", "4", "5", "6", "7" ]
