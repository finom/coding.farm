Balalaika DOM utilite

Balalaika is tiny (999 bytes uncompressed) embedded library that makes easier vanilla.js (pure JS) coding. This is individual project and will be presented on github soon.

It's using in Matreshka framework as DOM utilite when jQuery is not defined.

Balalaika extends Array. It means that you can use Array methods such as .forEach, .map, .reduce, .splice and so on.

Balalaika includes 3 own methods: .on, .off, .is and static .extend method that work same way as jQuery analogues (.on doesn't support data).

WARNING: This lib works fine in modern browsers (including Internet Explorer 9) only or using DOM shims. For older browsers you still need jQuery.

1.0

Class(prototype) → {function}

Javascript class implementation

Class function provides possibility to use classical OOP. Modern browsers use prototype based implementation but Internet Explorer 8 uses XDomainRequest hack to provide possibility to use getters and setters by Object.defineProperty method. Each time when you create an instance, the Class function core creates XDomainRequest instance and extends it by given prototype.
Returns: function (constructor)
Arguments
prototype object Methods and properties

xclass.js, line 28
Example

Usage

var A = Class({
	method1: function() { ... }
});
var B = Class({
	'extends': A,
	method2: function() { ... }
});
var C = Class({
	'extends': B,
	method2: function() {
		// "this" is execution context
		// arguments is just standard arguments pseudo array
		// the same as B.prototype.apply( this, arguments );
		C.parent.method2( this, arguments );
	},
	method3: function( a, b ) { ... }
});
var D = Class({
	'extends': C,
	method3: function( a, b ) {
		// you can pass any arguments to the method
		// the same as C.prototype.call( this, a, b );
		C.parent.method2( this, a, b );
	}
});

Type Definitions

arrayCallback(item, key, array)

Array callback is using in some Matreshka.Array methods like: Matreshka.Array#forEach, Matreshka.Array#filter, Matreshka.Array#some... as iterator callback.

To keep methods as fast as possible and reduce errors, most of methods are written using native Array.prototype methods. Pay attention that callback gets third argument that doesn't match self (this of the method). This is Array representation of Matreshka.Array instance because Array.prototype[ method ].apply( this, arguments ); doesn't work with DOM objects (XDomainRequest hack that we're using to keep dynamic accessors support). So we convert Matreshka.Array to Array and use it this way: Array.prototype[ method ].apply( representation, arguments );

key string

binder object

binder contains information about how to extract value from an element, how to set value for an element and which element's event we have to listen
Properties
on <optional>
 		string event name (or space-delimited list of events) which we have to listen
getValue <optional>
 		function function that tells how to extract value from an element (context this is given element)
setValue <optional>
 		function "How to set value" for an element (context this is given element)
Source:
matreshka-core.js, line 1522
Example
var binder = {
	on: 'click',
	getValue: function() { return this.value; } 
	setValue: function( v ) { this.value = v; } 
};
this.bindElement( 'a', '.my-checkbox', binder );

eventOptions object

eventOptions object could contain any properties

The one of special properties is "silent" that could be passed to Matreshka#set, Matreshka#remove, Matreshka#bind, Matreshka#unbind if you'd like to prevent the event from being triggered
Source:
matreshka-core.js, line 1524
Examples
var eventOptions = { silent: true };
this.a = 1;
this.on( 'change:a', function() { alert( 'a is changed' ); });
this.a = 2; // no alert
var eventOptions = { f: 'yeah' };
this.a = 1;
this.on( 'change:a', function( eventOptions ) { alert( eventOptions.f ); });
this.set( 'a', 2 ); // alerts "yeah"
