arrayCallback(item, key, array)

Array callback is using in some Matreshka.Array methods like: Matreshka.Array#forEach, Matreshka.Array#filter, Matreshka.Array#some... as iterator callback.

To keep methods as fast as possible and reduce errors, most of methods are written using native Array.prototype methods. Pay attention that callback gets third argument that doesn't match self (this of the method). This is Array representation of Matreshka.Array instance because Array.prototype[ method ].apply( this, arguments ); doesn't work with DOM objects (XDomainRequest hack that we're using to keep dynamic accessors support). So we convert Matreshka.Array to Array and use it this way: Array.prototype[ method ].apply( representation, arguments );

Parameters:
Name Type Description
item any
key string
array Array

Examples
var callback = function( item, key, array ) {
	console.log( item, key, array );
}
this.forEach( callback );

Third argument issue

this.forEach( function( item, key, array ) {
	console.log( this === array ); // false
}, this );

elementOptions

elementOptions contains information how to extract value from the element, how to set value for an element and which element event we have to listen
Type:
  • object
Properties:
Name Type Argument Description
on string <optional>
 event name (or events separated by space) on which we have to listen
getValue function <optional>
 function that tells to JS interpreter how to extract value from the element (context "this" is given element)
setValue function <optional>
 "How to set value" for the element (context "this" is given element)
Source:
  • matreshka.js, line 1163
Example
var elementOptions = {
	on: 'click',
	getValue: function() { return this.value; } 
	setValue: function( v ) { this.value = v; } 
};
this.bindElement( 'a', '.my-checkbox', elementOptions );

eventOptions

eventOptions could contain any properties. The only special property is "silent" that passed to Matreshka#set, {@link Matreshka#remove}, Matreshka#bind, {@link Matreshka#unbind} and says "Hs event has to be triggered after the method is executed?"
Type:
  • object
Source:
  • matreshka.js, line 1165
Examples
var eventOptions = { silent: true };
this.a = 1;
this.on( 'change:a', function() { alert( 'a is changed' ); });
this.a = 2; // no alert
var eventOptions = { f: 'yeah' };
this.a = 1;
this.on( 'change:a', function( eventOptions ) { alert( eventOptions.f ); });
this.set( 'a', 2 ); // alerts "yeah"
