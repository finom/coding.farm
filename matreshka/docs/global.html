$b()

Balalaika DOM utilite

Balalaika is tiny (999 bytes uncompressed) embedded library that makes easier vanilla.js (pure JS) coding. This is individual project and will be presented on github soon.

It's using in Matreshka framework as DOM utilite when jQuery is not defined.

Balalaika extends Array. It means that you can use Array methods such as .forEach, .map, .reduce, .splice and so on.

Balalaika includes 3 own methods: .on, .off, .is and static .extend method that work same way as jQuery analogues (.on doesn't support data).

Version:
1.0
Since:
0.1
Source:
balalaika.js, line 11
Examples
 $b( 'div' ).forEach( ... );
 $b( 'span', document.body ).map( ... );
 $b( '.button' ).on( 'click.mynamespace', ... );
 $b( '.button' ).off( 'click.mynamespace' );

retuire.config({
	paths: {
		'xclass': 'path/to/matreshka.min',
		'matreshka': 'path/to/matreshka.min',
		'balalaika': 'path/to/matreshka.min'
	}
});
require(['xclass', 'matreshka', 'balalaika'], function(Class, MK, $) {
	var $divs = $( 'div' );
	var MyClass = Class({
		'extends': MK
	});
});

AMD way (unnamed Matreshka module)

// Matreshka contains Class function as "Class" property and balalaika as "$b" property
require(['path/to/matreshka.min'], function(MK) {
	var $divs = MK.$b( 'div' );
	var MyClass = MK.Class({
		'extends': MK
	});
});

Class(prototype) → {xclass}

Javascript class implementation

Class function provides possibility to use classical OOP. Modern browsers use prototype based implementation but Internet Explorer 8 uses XDomainRequest hack to provide possibility to use getters and setters by Object.defineProperty method. Each time when you create an instance, the Class function core creates XDomainRequest instance and extends it by given prototype.
Returns: xclass (class (or rather costructor of a class))
Arguments
prototype object Methods and properties
Source:
xclass.js, line 36
Examples

Usage

var A = Class({
	method1: function() { ... }
});
var B = Class({
	'extends': A,
	method2: function() { ... }
});
var C = Class({
	'extends': B,
	method2: function() {
		// "this" is execution context
		// arguments is just standard arguments pseudo array
		// next string does the same thing as B.prototype.apply( this, arguments ); does
		C.parent.method2( this, arguments );
	},
	method3: function( a, b ) { ... }
});
var D = Class({
	'extends': C,
	method3: function( a, b ) {
		// you can pass any arguments to the method
		// next string does the same thing as C.prototype.call( this, a, b ); does
		C.parent.method2( this, a, b );
	}
});

Type Definitions

arrayCallback(item, key, array)

Array callback is using in some Matreshka.Array methods like: Matreshka.Array#forEach, Matreshka.Array#filter, Matreshka.Array#some... as iterator callback.

To keep methods as fast as possible and reduce errors, most of methods are written using native Array.prototype methods. Pay attention that callback gets third argument that doesn't match self (this of the method). This is Internet Explorer 8 support restriction and will be fixd after we refuse it.

Arguments
item any
key string
array Array
Source:
matreshka-array.js, line 1039
Examples
var callback = function( item, key, array ) {
	console.log( item, key, array );
}
this.forEach( callback );

Third argument issue

this.forEach( function( item, key, array ) {
	console.log( this === array ); // false
}, this );

binder object

binder contains information about how to extract value from an element, how to set value for an element and which element's event we have to listen
Properties
on <optional>
 		string function event name or space-delimited list of events or which we have to listen. Function contained callback is also supported.
getValue <optional>
 		function function that tells how to extract value from an element (context this is given element)
setValue <optional>
 		function "How to set value" for an element (context this is given element)
initialize <optional>
 		function Fuction that executes once per binding
Source:
matreshka-core.js, line 1816
Examples
var binder = {
	on: 'click',
	getValue: function( options ) { return this.value; } 
	setValue: function( v, options ) { this.value = v; },
	initialize: function( options ) {
		alert( 'Binder is initialized. Initial input value=' + this.value );
	}
};
this.bindElement( 'a', '.my-checkbox', binder );

Function as on property

var binder = {
	on: function( callback ) {
		this.onclick = callback;
	},
	getValue: function( options ) { return this.value; } 
	setValue: function( v, options ) { this.value = v; },
	initialize: function( options ) {
		alert( 'Binder is initialized. Initial input value=' + this.value );
	}
};
this.bindElement( 'a', '.my-checkbox', binder );

Examples

Space-delimited list of event names

this.on( 'x y z', function() {} );

change:KEY event. Listens changes of given property.

this.on( 'change:x', function( evt ) {
	alert( evt.value );
});
this.x = 1;

DOM events for bound elements (DOM_EVENT::KEY). Fires when fiven DOM event is fired on bound element (since v0.1.0).

this.bindElement( 'x', '.my-button' );
this.bindElement( 'y', '.my-select' );
this.on( 'click::x change::y', function( evt ) {
	alert( 'clicked "x" or changed "y"' );
});

Bubbling events (KEY@EVENT_NAME). You can attach event to any inner member of Matreshka (since v0.2.0). This event type is "live" and fires even when property is set after event initialized.

this.on( 'x@change:a', function() {
	alert( '"x.a" is changed' );
});
this.x = new MK;
this.x.a = 1;

Bubbling events (@EVENT_NAME) for MK.Object members (since v0.2.0). This event type is "live" and fires even when property is set after event initialized.

this.on( '@change:a', function() {
	alert( '"a" is changed' );
});
this.jset( 'x', new MK );
this.x.a = 1;

Bubbling events (@EVENT_NAME) for MK.Array members (since v0.2.0). This event type is "live" and fires even when element is added after event initialized.

this.on( '@change:a', function() {
	alert( '"a" is changed' );
});
this.push( new MK );
this[ 0 ].a = 1;

eventOptions object

eventOptions object could contain any properties

The one of special properties is "silent" that could be passed to Matreshka#set, Matreshka#remove, Matreshka#bind, Matreshka#unbind if you'd like to prevent the event from being triggered
Source:
matreshka-core.js, line 1818
Examples
var eventOptions = { silent: true };
this.a = 1;
this.on( 'change:a', function() { alert( 'a is changed' ); });
this.a = 2; // no alert
var eventOptions = { f: 'yeah' };
this.a = 1;
this.on( 'change:a', function( eventOptions ) { alert( eventOptions.f ); });
this.set( 'a', 2 ); // alerts "yeah"
