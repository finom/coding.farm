Matreshka

Matreshka is the core class that provides magical possibilities. It contains events engine, data-UI bindings engine, dependences engine and so on. Examples below shows main uses of Matreshka by providing source code and running code. Other examples (for each method) you can find on documentation page

Element binding

The next 3 examples show you how to bind data to UI. More info about bindElement method you can find in on documentation page. 

var mk = new MK(); // same as new Matreshka;
mk.bindElement( 'x', '.my-select', {
	on: 'change',
	getValue: function() {
		return this.value;
	},
	setValue: function( v ) {
		this.value = v;
	}
});

mk.x = 3;

	<option value="2">Second Option</option>
	<option value="3">Third Option</option>
	<option value="4">Fourth Option</option>
	<option value="5">Fifth Option</option>
</select>

Events

Matreshka includes powerful event engine. You can listen and trigger any event you want. Check out methods that do it: . 

var mk = new MK(); // same as new Matreshka;

mk.on( 'yeah', function() {
	var args = [].slice.call( arguments ).join( ', ' );
	$( '.output' ).html( args );
});

mk.trigger( 'yeah', 1, 'string', {} );

change: event

Matreshka allows you to listen property change event. When you're adding listener for change:%PROPERTY_NAME% event, Matreshka creates setter for this property. It means that you can change the property directly, without calling methods such as .set.

var mk = new MK();

mk.on( 'change:x', function() {
	var text = 'x changed to ' + this.x;
	$( '.output' ).html( text );
});

mk.x = 5;

<div class="output"></div>

.setMediator method

This method allows you to keep property valid and keep it to be a string/number/function/object... even if incoming value has another type. This logic is very similar to input value (if you're sessing input value it always converts to string).

Get more on documentation page

var mk = new MK();
mk.setMediator({
	percentageValue: function( v ) {
		return v > 100 ? 100 : v < 0 ? 0 : v;
	},
	stringValue: String,
	integerValue: parseInt
});

mk.on( 'change', function() {
	$( '.output' ).html(
		'percentageValue = ' + JSON.stringify( this.percentageValue )
		+ '<br>stringValue = ' + JSON.stringify( this.stringValue )
		+ '<br>integerValue = ' + JSON.stringify( this.integerValue )
	);
});

mk.percentageValue = -5;
mk.stringValue = 1234;
mk.integerValue = '6789.123456';

.addDependence method

.addDependence method allows property to be dependent from other properties.

Get more on documentation page

var mk = new MK();
mk.on( 'change:a', function() {
	$( '.output' ).html( 'a = ' + this.a );
});

// a depends from b and c
mk.addDependence( 'a', 'b c', function() {
	return this.b + this.c;
});

mk.b = 11;
mk.c = 22;

MK.Object

After you got how to work with Matreshka you need to delimit application data from application state. What does it mean? For example, you're developing simple login form. It should have two inputs (login and password), "Show Password" checkbox, "Remember Me" checkbox and submit button. All of these elements represent "application state". Server needs 3 things: login, password, and "Remember Me" option, but it doesn't need to know is password is showing to user or not. So we have login, password and "Remember Me" as data state and login, password, "Remember Me" and "Show Password" as application state. As you see data state is always a subset of application state. Check this example out on JSBin.

Documentation page

.jset and .toJSON

How to add properties to appliaton data set? It could be done using jset method by passing needed properties to this method once. From now you don't need to be care which properties are responsible for data and you can work with any data property as with regilar property. Look at docs: jset, toJSON, keys, addJSONKeys, removeJSONKeys.

var     mkObject = new MK.Object,
	result;

// making 'a' and 'b' data keys
mkObject.jset({
	a: 1,
	b: 2
});

mkObject.b = 3;

// c is not data so it's not part of result
// of the .toJSON method
mkObject.c = 4;

result = mkObject.toJSON();

$( '.output' ).html( JSON.stringify( result ) );

MK.Array

MK.Array is responsible for collections in Matreshka apps. It inherits Matreshka class and has same features as Matreshka (events, dependences etc) but provides collection methods.

Documentation page

Original Array methods

Remember that Matreshka.Array instance has same methods as native Javascript Array. Some methods are changed for chain calling (sugared native Array methods).

var mkArray = new MK.Array(),
	result;
mkArray
	.push( 1, 2, 3, 4, 5 )
	.shift()
;
result = mkArray
	.unshift( 6, 7 )
	.splice( 1, 4 )
	.toString()
;				
$( '.output' ).html( result );

Events

Each method that modifies Matreshka.Array instance fires two events: method name and "modify". Some of methods that add or remove (or both: add and remove) fire corresponding events. For example .push method fires "push", "modify" and "add" events, .sort method fires "sort"and "modify" events, .splice method fires "splice", "modify", "add" (if something is added) and "remove" (if something is remove) events.. and so on. Event options that passed to methods contain information about what is going on (what is added/removed, which method is called...).

var mkArray = new MK.Array();

mkArray
	.on( 'modify', function( evtOpts ) {
		alert(
			'"modify" event is fired\nresult is: ' 
			+ evtOpts.returns + '\narguments: ' 
			+ ( [].slice.call( evtOpts.args ).toString() || 'none' )
			+ '\nfired by: .' + evtOpts.method + ' method'
		);

		$( '.output' ).html( this.join( ', ' ) );
	})
	.on( 'push', function( evtOpts ) {
		alert(
			'"push" event is fired\nresult is: ' 
			+ evtOpts.returns 
			+ ' (original Array.prototype push returns length)\narguments: ' 
			+ [].slice.call( evtOpts.args ) 
		);
	})
	.on( 'pop', function( evtOpts ) {
		alert(
			'"pop" event is fired\nresult is: ' 
			+ evtOpts.returns 
			+ ' (original Array.prototype.pop returns last array item)'
		);
	})
	.on( 'reverse', function( evtOpts ) {
		alert( '"reverse" event is fired' );
	})
	.on( 'sort', function( evtOpts ) {
		alert( '"sort" event is fired' );
	})
;

$( '.push-example' ).on( 'click', function() {
	mkArray.push( 1, 2, 3, 4, 5 );
});
$( '.pop-example' ).on( 'click', function() {
	mkArray.pop();
});
$( '.reverse-example' ).on( 'click', function() {
	mkArray.reverse();
});
$( '.sort-example' ).on( 'click', function() {
	mkArray.sort();
});
